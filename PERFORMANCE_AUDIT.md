# Performance, bfcache, and Service Worker Audit

Audit date: 2026-06-04
Site version: v0.18.3
Preview target: `http://127.0.0.1:4321/`
Canonical command: `npm run audit:perf`

## Summary

The portfolio has an explicit service-worker update prompt instead of silently activating a new worker over an open tab. The repeatable local Chromium audit documents the representative homepage, search page, archive page, project detail page, and desktop homepage samples for LCP, CLS, lab event timing, horizontal overflow, console/network errors, and bfcache restore.

The v0.18.3 run shows clean LCP, CLS, event timing, bfcache restore, overflow, and console/network status across all samples. The prior mobile homepage LCP warning is resolved after inlining `src/styles/critical.css` for the nav/hero first viewport, loading the full hashed `global.css` bundle with a preload plus print-media swap, and reintroducing guarded `content-visibility:auto` for named below-fold homepage sections.

The audit also caught two project-page regressions before this item closed: raw HTML README images such as `logo.png` were requesting missing same-origin files, and wide README/project header content could expand the mobile viewport. Project pages now rewrite raw README image sources to GitHub raw URLs and constrain mobile project-detail content to the viewport.

## Method

`scripts/audit-performance.mjs` launches an isolated local Chrome or Edge profile through the Chrome DevTools Protocol and writes ignored JSON to `.tmp/performance-audit.json`.

The script records:

- LCP through `PerformanceObserver` for `largest-contentful-paint`.
- CLS through buffered `layout-shift` entries, excluding recent-input shifts.
- Lab interaction timing through buffered `event` entries after scripted scroll/keyboard probes.
- Long tasks through buffered `longtask` entries.
- bfcache restore by navigating away and checking `pageshow.persisted` after browser history back.
- Horizontal overflow by comparing document scroll width to client width.
- Console warnings/errors, failed requests, and HTTP 4xx/5xx responses.

By default, threshold misses are printed as `WARN` so the audit can be used during documentation runs. Add `--strict` when threshold warnings should fail the command.

INP is a field metric and cannot be proven by this local lab run. The `max event` column below is a local Event Timing proxy for INP risk, not real-user INP.

## Thresholds

- LCP: `<= 2500ms`.
- CLS: `<= 0.1`.
- Lab max event duration: `<= 200ms`.
- bfcache: `pageshow.persisted` must be observed after history back.
- Horizontal overflow and console/network errors must be absent.

## Service Worker Update UX

Local implementation:

- `public/sw.js` uses a `portfolio-v__BUILD_VERSION__` placeholder that `npm run sw:stamp` stamps to the package version in `dist/sw.js` during build.
- The install handler precaches without unconditional `skipWaiting()`.
- `public/sw.js` accepts a `{ type: "SKIP_WAITING" }` message.
- `public/scripts/main.js` registers the worker on page load, listens for `updatefound`, detects an installed waiting worker, and shows an accessible update toast.
- The page reloads on `controllerchange` only after the visitor chooses the toast refresh action.

This avoids silent refreshes while preserving static offline caching.
