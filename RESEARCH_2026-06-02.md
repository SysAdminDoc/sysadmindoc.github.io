## Executive Summary

The portfolio is a mature, privacy-first static Astro 6 site for a senior systems-administrator / medical-imaging-IT builder. The v0.18.0 drain (≈85 items) hardened SEO/AEO (connected JSON-LD graph, llms.txt, JSON Feed, sitemap), accessibility (forced-colors, landmarks, terminal a11y), PWA (install prompt, SWR service worker), feeds, and a major perf win (44KB inline command-palette data → one cached `/cmdk-data.js`). The single most important finding of this pass was a **P0 regression that left the entire homepage interactive layer dead** — now fixed in v0.18.1. With that resolved, the highest-value direction is **closing the gap between the site's strong evidence assets and what a skim-reading recruiter actually sees** (surface `proof.ts` outcomes above the fold, fix data-consistency flicker), plus **finishing the AEO depth on interior routes** and **converting the standing mobile-LCP issue into a measured, critical-CSS fix**.

**Top opportunities, priority order:**

1. **[DONE this pass] P0 — `main.js` ran before `shared.js`** → `ReferenceError`, killing the terminal, catalog search/filter/sort, live stars, PWA install, and SW registration. Fixed + guarded + shipped (v0.18.1). **Verified live.**
2. **P1 — Surface quantified outcomes above the fold.** `proof.ts` holds recruiter-grade results (1M+ file migration, 54 cloud accounts restored, production adoption) but they're buried two clicks deep.
3. **P1 — Page-level JSON-LD on the 8 interior routes** (esp. a ProfilePage/Person on `/resume`, which already ships `resume.json` but no page schema) + a `SoftwareApplication` node for the 22 live apps.
4. **P1 — Data-consistency fixes:** hero "176+ projects" vs catalog "181"; the language-donut count/percentages visibly shift between build-time and JS render; skill-ring fills contradict the real category distribution.
5. **P1 — Greatest Hits case-study coverage is 3/8** — the strongest hiring asset is half-wired; 5 of 8 flagship cards fall back to a thin README.
6. **P1 — Mobile LCP: critical-CSS inline, not `@layer`.** Two render-blocking stylesheets (~118KB) gate the hero `<h1>`; re-baseline the stale `PERFORMANCE_AUDIT.md` (v0.16.12) first.
7. **P1 — forced-colors gap:** heatmap, language donut, and skill rings (SVG data-viz) are unreadable in Windows High Contrast.
8. **P2 — CI realism + gates:** PR gate stubs all data to empty (never tests real shapes); a11y gate is advisory-by-construction; `deploy.yml` runs no a11y/test step.
9. **P2 — README TOC + reading-time** on project pages (heading IDs are already generated but orphaned; 167 repos have READMEs, several 4,000+ words).
10. **P2 — WebSite `SearchAction`** (sitelinks search box) + README-content surfacing as the honest "content engine" (a TIL feed stays correctly parked per `NOTES_FEED_POLICY.md`).

## Highest-Value New Features

### NF-A1 — Above-the-fold "proof strip" of quantified outcomes · P1 · M · Verified evidence
- **Problem:** A skim-reading hiring manager never sees the strongest evidence; the quantified outcomes live one click deep in project pages.
- **Evidence:** `proof.ts` `caseStudy.outcomes` hold concrete results (win11-nvme "used internally on Maven Imaging workstations… measurably reduced study load times"; Network_Security_Auditor; NovaCut). Hero `heroSignals` (`index.astro:114-130`) are role/scope only; the million-file migration appears only as prose in About (`index.astro:599`).
- **Proposed:** A compact, build-time `ProofStrip` between Greatest Hits and Live Apps pulling 3–4 outcomes from `proof.ts` (e.g. "1M+ files migrated", "54 cloud accounts recovered", "shipped to company-wide production"). Data-driven so it can't drift.
- **Touches:** `src/data/proof.ts` (or a derived outcomes list), `src/pages/index.astro`, optional `src/components/ProofStrip.astro`.
- **Verify:** Strip renders in the first 1–2 viewports with numbers sourced from `proof.ts`; no hand-duplicated literals.

### NF-A2 — Page-level JSON-LD on the 8 interior routes (AEO depth) · P1 · M · Verified
- **Problem:** AI/rich-result crawlers see only the generic `WebSite+Person` graph on `/now`, `/uses`, `/resume`, `/timeline`, `/releases`, `/archive`, `/healthcare-it`, `/search`. The resume page ships a full `resume.json` but no page schema tying it to a Person/occupation.
- **Evidence:** Extracted `@type` per built page: those 8 routes → `[WebSite,Person]` only; contrast `lang/python` (CollectionPage+Breadcrumb) and `projects/*` (SoftwareSourceCode+Breadcrumb).
- **Proposed:** Add per-page nodes + BreadcrumbList reusing the `#matt-parker` `@id`: `ProfilePage`/`Person`-with-`Occupation` on `/resume`, `AboutPage` on `/healthcare-it`, `CollectionPage`/`ItemList` on `/releases` and `/timeline`, `WebPage` on `/now` and `/uses`.
- **Touches:** `src/layouts/Base.astro` (optional `pageJsonLd` prop) or per-page `<script type="application/ld+json">` mirroring `lang/[slug].astro`.

### NF-A3 — `SoftwareApplication` node + license/datePublished for live apps · P1 · M · Verified

Capture `licenseSpdx`+`createdAt` in `fetch-stars.mjs`.
- **Touches:** `src/pages/projects/[slug].astro` (~line 287 jsonLd), `scripts/fetch-stars.mjs` (`_meta.json` shape — update `generated.d.ts`).
- **Verify:** A live-app page shows a `SoftwareApplication` with `offers.price=0`; Rich Results test passes.

## Reliability, Security, Privacy, Data Safety

- **[RESOLVED] P0 homepage interactivity** — fixed in v0.18.1 (script order + build guard + SW cache bump). The guard in `fix-html-structure.mjs` now fails the build if `main.js` ever precedes `shared.js`. **Verified live.**
- **Astro 6 `</html>` band-aid (P2 debt):** `fix-html-structure.mjs` regex-rewrites every dist page each build to repair Astro 6.3.3 emitting `</html>` after `</head>` (0-indexes Pagefind). Two workarounds exist for one bug (the script + JSON-LD-in-`<body>`). **Action:** bisect with `compressHTML:false` (prime suspect; `astro.config.mjs:39`) and the prefetch/clientPrerender head-injected script; if isolated, drop the post-build step + the body-JSON-LD workaround, else file/track upstream and pin. *(Note: my session's guard already makes a silent regression fail loudly.)*
- **CI never exercises real data shapes (P2 test-gap):** `ci.yml:29-39` stubs `_*.json` to empty `{}`/`[]`, so build/check/a11y run against a degenerate site (no projects, no README HTML). Real-data bugs (sanitize-html edges, populated donut/heatmap, JSON-LD arrays) can't be caught pre-merge. **Action:** commit a tiny `src/data/_fixtures/` (3–4 projects + 1 README + 1 release), copy into `_*.json` in CI.
- **deploy.yml runs no a11y/test step (P2):** a direct push to `main` (bypassing PR) ships without the a11y audit or `npm test`. Mirror `npm test` + strict a11y into `deploy.yml`'s build job.
- **CSP `unsafe-inline` (P3 partial win):** required by the inline theme-init + `define:vars`. Full removal is deferred (T95), but the theme-init (`Base.astro:125`) is one small hashable block — a hash/nonce for just that script is a smaller partial win than full T95.

## UX, Accessibility, Trust

- **forced-colors SVG data-viz gap (P1):** heatmap intensity (`.hm-0..hm-4` rgba greens, `global.css:3656-3686`), language donut strokes, skill rings, and the conic-gradient fallback ring are unreadable/invisible in Windows High Contrast (forced-colors doesn't adjust SVG fill/stroke or gradients). **Action:** in `@media(forced-colors:active)` map buckets/segments to system-color pairs (Highlight/CanvasText), replace the conic-gradient fallback with a bordered placeholder, and ensure the textual legends aren't `aria-hidden`. *(Video-close button and install chip are already forced-colors-safe — real `<button>` + currentColor.)*
- **prefers-contrast unsupported (P2):** zero `prefers-contrast` rules; macOS/iOS/Android "Increase Contrast" users get no enhancement for the many low-alpha borders/`--t3` text. Add a lightweight `@media(prefers-contrast:more)` token override.
- **a11y gate is advisory-by-construction (P2):** `ci.yml:46-47` wraps the audit in `|| echo`, and `audit-a11y.mjs` exits 0 without `--strict` — it can never fail a PR. Promote the 6 zero-false-positive rules to a blocking `--strict` step (drop the `||`).
- **No axe/Playwright a11y on hydrated widgets (P2):** the regex audit can't see computed contrast, ARIA relationships, or JS-built widgets (terminal `role=log`, cmdk listbox, install chip, video close — all shipped this cycle). Add an `@axe-core/playwright` job (Playwright is already an optional dep).

## Architecture and Maintainability

- **Mobile LCP — critical-CSS is the lever, not `@layer` (P1):** the home page links two render-blocking sheets — `Base.*.css` (110KB) + `index.*.css` (8.4KB) — and `inlineStylesheets:'auto'` (4KB threshold) inlines neither, so ~118KB downloads before the hero `<h1>` paints. **`@layer` (deferred T16) reorders cascade but does NOT reduce blocking bytes** — note this in the T16 rationale so it isn't mistaken for a perf fix. **Action:** inline a small hero/nav critical block in `<head>` and load the remainder non-blocking.
- **Re-baseline `PERFORMANCE_AUDIT.md` (P1, S):** dated 2026-05-17 / **v0.16.12**, predating the cmdk extraction (the biggest perf win). Run `npm run audit:perf` against v0.18.1 and update the table; the 3156ms mobile-LCP "standing issue" may already be partly resolved.
- **Visual-regression coverage (P2):** zero pixel/DOM-snapshot net — the precise reason T16 (`@layer`) and T28 (`<Picture>`) are design-gated. A Playwright `toHaveScreenshot` job (`/`, `/search`, a project page, `/archive` × mobile+desktop, baselines in CI/Docker) **unblocks both deferred items**.
- **Duplicated career literals:** "10+ PACS migrations" / "15+ years" hardcoded in multiple places (`index.astro:122-123,266,599`); candidate for a single `careerFacts` source alongside `career.ts`.
- **Catalog DOM ceiling (P3):** all 181 entries server-rendered; sort re-appends the whole grid, filter toggles every node. Fine now; add a DOM-size/HTML-byte budget gate so growth is visible before ~300 entries warrant windowing.
